## Authentication

All API endpoints require authentication. You can use either **JWT tokens** or **API secrets**:

**JWT Tokens:**
```bash
Authorization: Bearer YOUR_JWT_TOKEN
```

**API Secrets (recommended for automation):**
```bash
Authorization: Bearer wai_abc123def456...
```

API secrets never expire and are ideal for server-to-server communication. See the [API Secrets documentation](/api-reference/api-secrets) for details.

## Base URL

All API requests should be made to:

```
https://waterr.ai/backend
```

## Response Format

Most API responses follow this format:

**Success Response:**
```json
{
  "status": "success",
  "data": { ... }
}
```

Some endpoints (users, meetings) use an alternate format:
```json
{
  "success": true,
  "data": { ... }
}
```

**Error Response:**
```json
{
  "status": "error",
  "message": "Error description"
}
```

## Rate Limits

API rate limits are applied per API token. Check response headers for current rate limit information:
- `X-RateLimit-Limit`: Maximum requests allowed
- `X-RateLimit-Remaining`: Remaining requests in current window
- `X-RateLimit-Reset`: Time when the rate limit resets
