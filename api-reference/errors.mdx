## Error Response Format

Most endpoints return errors in this format:

```json
{
  "status": "error",
  "message": "Scenario not found"
}
```

Some endpoints (users, meetings, session limits) use an alternate format:

```json
{
  "success": false,
  "error": "Active workspace not found."
}
```

## HTTP Status Codes

| Code | Meaning | When you'll see it |
|------|---------|-------------------|
| `200` | OK | Successful GET, PUT, or PATCH |
| `201` | Created | Successful POST (resource created) |
| `204` | No Content | Successful DELETE |
| `400` | Bad Request | Missing required fields, invalid field values, malformed JSON |
| `401` | Unauthorized | Missing or invalid `Authorization` header, expired JWT token, revoked API secret |
| `403` | Forbidden | Valid auth but insufficient permissions (e.g., accessing another org's data) |
| `404` | Not Found | Resource doesn't exist, or wrong ID |
| `409` | Conflict | Duplicate resource (e.g., persona name already exists in org) |
| `422` | Unprocessable Entity | Valid JSON but business logic rejection (e.g., invalid demeanor value) |
| `429` | Too Many Requests | Rate limit exceeded -- see [Rate Limits](/api-reference/rate-limits) |
| `500` | Internal Server Error | Something broke on our end -- retry or contact support |

## Common Errors and Fixes

### Authentication errors

```json
{ "status": "error", "message": "No token provided" }
```
**Fix:** Add the `Authorization: Bearer YOUR_TOKEN` header.

```json
{ "status": "error", "message": "Invalid or expired token" }
```
**Fix:** Your JWT has expired. Use an [API secret](/api-reference/authentication) instead (they never expire), or re-authenticate to get a fresh JWT.

### Resource errors

```json
{ "status": "error", "message": "Scenario not found" }
```
**Fix:** Check the ID. IDs are UUIDs (e.g., `a1b2c3d4-e5f6-...`), not integers.

### Validation errors

```json
{ "status": "error", "message": "Validation error", "details": "name cannot be null" }
```
**Fix:** Check required fields in the endpoint documentation. `name`, `type`, and `persona_id` are required for scenarios.

### Session limit errors

```json
{ "success": false, "error": "Insufficient session minutes" }
```
**Fix:** Your organization has used all session minutes for this billing period. Upgrade your plan or wait for the next period.

## Debugging Tips

<CardGroup cols={2}>
  <Card title="Check the status code first" icon="circle-info">
    The HTTP status code tells you the category of error before you even read the message.
  </Card>
  <Card title="Log full responses" icon="bug">
    Always log the complete response body, not just the status code. The `message` and `details` fields contain the fix.
  </Card>
  <Card title="Use API secrets for automation" icon="key">
    JWT tokens expire. If your integration breaks overnight, switch to [API secrets](/api-reference/authentication).
  </Card>
  <Card title="IDs are UUIDs" icon="fingerprint">
    All resource IDs are UUIDs, not integers. Double-check you're not passing a numeric ID.
  </Card>
</CardGroup>
