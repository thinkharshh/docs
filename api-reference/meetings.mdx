## Overview

A meeting is a live AI conversation session created from a scenario. Creating one spins up a Daily.co video room with your AI persona ready to talk and returns the room URL and token.

## Create Meeting

Create a new meeting from a scenario. This is the main endpoint you call to start a session for a participant.

<ParamField body="person_name" type="string" required>
  Name of the participant joining the session
</ParamField>

<ParamField body="scenario_id" type="string" required>
  UUID of the scenario to use
</ParamField>

<ParamField body="voice_id" type="string">
  Override the persona's default voice for this session. Get available voices from the [Voices API](/api-reference/voices).
</ParamField>

<ParamField body="meeting_type" type="string">
  `regular` (default) or `instant`
</ParamField>

<ParamField body="note_context" type="string">
  Session-specific context injected into the AI's prompt without changing the scenario. Use this to personalize: "Candidate has 8 years at Google, specializes in ML infrastructure."
</ParamField>

```bash
curl -X POST https://waterr.ai/backend/meetings \
  -H "Authorization: Bearer YOUR_API_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "person_name": "John Doe",
    "scenario_id": "scenario-uuid",
    "note_context": "Candidate previously worked on distributed caching at scale. Focus on consistency vs availability trade-offs."
  }'
```

### Response

```json
{
  "success": true,
  "data": {
    "id": "meeting-uuid",
    "daily_meeting_id": "abc123",
    "daily_meeting_url": "https://waterr-xx.daily.co/abc123",
    "room_token": "eyJhbGciOiJIUzI1NiIs...",
    "meeting_url": "https://waterr.ai/meeting/meeting-uuid",
    "person_name": "John Doe",
    "scenario_id": "scenario-uuid",
    "status": "created",
    "meeting_type": "regular",
    "scenario": {
      "name": "System Design Interview",
      "type": "interview",
      "description": "Practice system design for senior backend roles"
    }
  },
  "session_minutes_info": {
    "estimatedMinutes": 25,
    "remainingMinutes": 75
  }
}
```

### Key response fields

| Field | Use |
|-------|-----|
| `daily_meeting_url` | The Daily.co room URL — embed this or redirect the participant here |
| `room_token` | Auth token for the Daily.co room (needed for SDK-based joins) |
| `meeting_url` | WaterrAI-hosted meeting page URL (participant-friendly) |
| `id` | Meeting UUID — use this to fetch analysis, transcript, and recording later |
| `session_minutes_info` | Your remaining session minutes after this meeting |

### Joining the meeting

**Option 1: WaterrAI hosted page** — redirect the participant to `meeting_url`:
```
https://waterr.ai/meeting/{meeting_id}
```

**Option 2: Embed with iframe**
```html
<iframe
  src="DAILY_MEETING_URL?t=ROOM_TOKEN"
  allow="camera; microphone; autoplay; display-capture"
  style="width: 100%; height: 600px; border: none;"
></iframe>
```

**Option 3: Daily.co SDK** (full control over UI)
```javascript
import DailyIframe from '@daily-co/daily-js';

const call = DailyIframe.createCallObject();
await call.join({
  url: response.data.daily_meeting_url,
  token: response.data.room_token,
});
```

## Create End-User Meeting

For public/embedded scenarios — no JWT required. Uses the `allowEmbed` middleware.

```bash
curl -X POST https://waterr.ai/backend/meetings/end-user \
  -H "Content-Type: application/json" \
  -d '{
    "person_name": "Jane Smith",
    "scenario_id": "scenario-uuid",
    "end_user_id": "end-user-uuid"
  }'
```

<ParamField body="end_user_id" type="string" required>
  UUID of the end user (from your auth flow). Session minutes are billed to the scenario creator's account.
</ParamField>

<Note>
  End-user meetings consume session minutes from the **scenario creator's** account, not the end user's. This is how embedded and public scenarios work.
</Note>

## End Meeting

Programmatically end a meeting (e.g., when a time limit is reached).

```bash
curl -X PUT https://waterr.ai/backend/meetings/{meetingId}/end \
  -H "Authorization: Bearer YOUR_API_TOKEN"
```

## Delete Meeting

Soft-delete a meeting (marks as deleted but preserves data).

```bash
curl -X PATCH https://waterr.ai/backend/meetings/{meetingId}/soft-delete \
  -H "Authorization: Bearer YOUR_API_TOKEN"
```

## What happens after a meeting ends

1. **Recording** is finalized and uploaded (~2-5 min)
2. **Transcript** is processed into sentences (~30 sec)
3. **Analysis** with goal scores is generated (~60 sec)
4. **Webhook** fires (if configured) with full results
5. **Email notifications** sent to configured recipients

See [Session Lifecycle](/api-reference/session-lifecycle) for the complete flow and how to poll or listen for results.
