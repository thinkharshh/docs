## Overview

The Users API provides endpoints to retrieve user workspace information. The most important endpoint is **Get Active Workspace**, which returns your membership ID required for creating meetings and other API calls.

<Warning>
  **Pre-Setup Required:** Before creating meetings or using other organization-scoped APIs, you must first retrieve your active workspace to get your `membership_id`.
</Warning>

## Get Active Workspace

Retrieve the current user's active workspace and membership information.

```bash
curl -X GET https://waterr.ai/backend/users/active-workspace \
  -H "Authorization: Bearer YOUR_API_TOKEN"
```

```json
{
  "success": true,
  "data": {
    "id": 42,
    "type": "creator",
    "org_id": 15,
    "user_basic_info_id": 123,
    "role": "admin",
    "active": true,
    "created_at": "2024-01-10T08:00:00Z",
    "Organization": {
      "id": 15,
      "name": "My Company",
      "created_at": "2024-01-10T08:00:00Z"
    }
  }
}
```

### Response Fields

| Field | Type | Description |
|-------|------|-------------|
| `id` | integer | **The membership ID** -- use this for creating meetings and other APIs |
| `type` | string | Membership type: `creator` or `end_user` |
| `org_id` | integer | Organization ID the user belongs to |
| `role` | string | User's role in the organization |
| `active` | boolean | Whether this membership is currently active |
| `Organization` | object | Organization details including name |

<Note>
  Save the `id` value from the response -- this is your `membership_id` that you'll need for creating meetings and other organization-scoped operations.
</Note>

## Usage Example

Here's a complete flow showing how to get your membership ID before creating a meeting:

```javascript
const API_BASE = 'https://waterr.ai/backend';
const API_TOKEN = 'YOUR_API_TOKEN';

async function createMeetingWithWorkspace() {
  // Step 1: Get active workspace
  const workspace = await fetch(`${API_BASE}/users/active-workspace`, {
    headers: { 'Authorization': `Bearer ${API_TOKEN}` }
  }).then(r => r.json());

  const membershipId = workspace.data.id;

  // Step 2: Create a meeting
  const meeting = await fetch(`${API_BASE}/meetings`, {
    method: 'POST',
    headers: {
      'Authorization': `Bearer ${API_TOKEN}`,
      'Content-Type': 'application/json'
    },
    body: JSON.stringify({
      person_name: "John Doe",
      scenario_id: 456,
      meeting_type: "regular",
      membership_id: membershipId
    })
  }).then(r => r.json());

  console.log('Meeting created:', meeting.data);
}
```

<Tip>
  Cache the membership ID after the first call -- it typically doesn't change unless the user switches organizations.
</Tip>

## Get User by ID

Retrieve a user's public profile by their user ID. No authentication required.

<ParamField path="userId" type="string" required>
  The user UUID
</ParamField>

```bash
curl -X GET https://waterr.ai/backend/users/{userId}
```

## Get User by Username

Look up a user by their username. No authentication required. Useful for building public profile pages or resolving usernames to IDs.

<ParamField path="username" type="string" required>
  The username
</ParamField>

```bash
curl -X GET https://waterr.ai/backend/users/username/{username}
```
