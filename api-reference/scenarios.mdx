## Overview

Scenarios define conversation contexts that combine a persona with instructions, goals, and settings. Scenarios can be public (for embedding) or restricted.

## Get All Scenarios

```bash
curl -X GET https://waterr.ai/backend/scenarios \
  -H "Authorization: Bearer YOUR_API_TOKEN"
```

## Get Scenario for Embed

Public endpoint — no authentication required for public scenarios.

```bash
curl -X GET https://waterr.ai/backend/scenarios/embed/{id}
```

<Note>
  Only works for scenarios with `visibility: "public"`. For restricted scenarios, use the authenticated GET endpoint.
</Note>

## Create Scenario

<ParamField body="name" type="string" required>
  Scenario name (shown in dashboard and to participants)
</ParamField>

<ParamField body="type" type="string" required>
  Category: `interview`, `roleplay`, `upskill`, `brainstorm`
</ParamField>

<ParamField body="persona_id" type="string" required>
  UUID of the persona to use
</ParamField>

<ParamField body="prompt" type="string">
  Meeting script — conversation flow, behavioral rules, guardrails. Markdown supported. See the [Prompting Guide](/api-reference/prompting-guide).
</ParamField>

<ParamField body="description" type="string">
  Short description shown on the join page
</ParamField>

<ParamField body="call_duration" type="integer">
  Maximum session length in minutes. Default: 30
</ParamField>

<ParamField body="welcome_message" type="string">
  AI's opening line when a participant joins. Auto-generated from prompt if omitted.
</ParamField>

<ParamField body="visibility" type="string">
  `public` (default) or `restricted`
</ParamField>

<ParamField body="goals" type="array">
  Array of [Goal](/api-reference/goals) UUIDs for post-session evaluation
</ParamField>

<ParamField body="user_prompt" type="string">
  Your original plain-text description (stored for reference)
</ParamField>

```bash
curl -X POST https://waterr.ai/backend/scenarios \
  -H "Authorization: Bearer YOUR_API_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "System Design Interview",
    "description": "Practice system design for senior backend roles",
    "type": "interview",
    "persona_id": "persona-uuid",
    "prompt": "You are Sarah Chen, VP of Engineering...\n\n## Flow\n1. Intro (2 min)\n2. System design problem (15 min)\n3. Follow-ups (5 min)\n4. Their questions (3 min)\n\n## Rules\n- One question at a time\n- Push back on vague answers",
    "call_duration": 25,
    "welcome_message": "Hi! I'\''m Sarah. Let'\''s dive into a system design discussion.",
    "goals": ["goal-uuid-1", "goal-uuid-2"]
  }'
```

<Note>
  Creating a scenario automatically generates a `MeetingControls` record with default settings (recording on, camera off, screen share off, etc.) and a unique `scenario_link` for sharing.
</Note>

## Create with AI (GPT)

Generate a complete scenario — persona, prompt, and goals — from a text description. Optionally attach a file (resume, job description, product brief).

```bash
curl -X POST https://waterr.ai/backend/scenarios/create-with-gpt \
  -H "Authorization: Bearer YOUR_API_TOKEN" \
  -F 'userInput=Sales discovery call with a skeptical enterprise buyer evaluating 3 competing solutions. 20 minutes.' \
  -F 'duration=20' \
  -F 'allow_live_vision=false' \
  -F 'allow_web_search=true'
```

<ParamField body="userInput" type="string" required>
  Plain-text description of what you want. Be specific about context, roles, and goals.
</ParamField>

<ParamField body="file" type="file">
  Resume, job description, or product brief (PDF, DOC, DOCX). The AI uses this as context.
</ParamField>

<ParamField body="duration" type="integer">
  Session duration in minutes. Default: 30
</ParamField>

<ParamField body="allow_live_vision" type="string">
  `"true"` or `"false"` — enable camera/screen vision
</ParamField>

<ParamField body="allow_web_search" type="string">
  `"true"` or `"false"` — enable web search during conversation
</ParamField>

<ParamField body="type" type="string">
  Override the AI-detected type
</ParamField>

**Response:**
```json
{
  "status": "success",
  "data": {
    "scenario_id": "generated-uuid",
    "scenario_type": "roleplay",
    "generatedPersona": {

      "avatar": "https://...",
      "voice_id": "..."
    },
    "createdGoals": [
      { "id": "...", "name": "Discovery Quality", "description": "..." },
```

## Update Scenario

<ParamField path="id" type="string" required>
  The scenario UUID
</ParamField>

```bash
curl -X PUT https://waterr.ai/backend/scenarios/{id} \
  -H "Authorization: Bearer YOUR_API_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Advanced System Design Interview",
    "call_duration": 30,
    "prompt": "Updated instructions..."
  }'
```

## Update Session Options

Control session behavior settings for a scenario.

```bash
curl -X PUT https://waterr.ai/backend/scenarios/{scenarioId}/session-options \
  -H "Authorization: Bearer YOUR_API_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "duration": 30,
    "booking_enabled": true,
    "booking_url": "https://cal.com/yourname/interview",
    "webhook_url": "https://your-app.com/webhooks/waterr"
  }'
```

## Delete Scenario

```bash
curl -X DELETE https://waterr.ai/backend/scenarios/{id} \
  -H "Authorization: Bearer YOUR_API_TOKEN"
```

## Meeting Controls

Every scenario has associated meeting controls that configure the session experience. These are created with defaults when you create a scenario. To update them, use the scenario settings endpoints.

| Control | Default | Description |
|---------|---------|-------------|
| `allow_recording` | `true` | Record the session |
| `allow_transcription` | `false` | Auto-transcribe audio |
| `allow_live_vision` | `false` | AI can see camera/screen |
| `allow_web_search` | `false` | AI can search the web |
| `allow_screen_share` | `false` | Participant can share screen |
| `allow_participant_video` | `false` | Participant camera |
| `allow_participant_audio` | `true` | Participant microphone |
| `deep_think` | `false` | AI pauses to reason before responding |
| `language` | `"English"` | Conversation language |
| `max_participants` | `null` | Max participants (null = unlimited) |
| `end_call_enabled` | `false` | Participant can end the call |
