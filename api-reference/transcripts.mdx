## Overview

Transcripts provide a structured record of meeting conversations.

<Note>
  **Recommended:** Use **sentence transcripts** (`/sentenced-transcripts/*`) for the best developer experience (speaker labels, ordering, timestamps).
</Note>

## Get All Sentence Transcripts for a Scenario

<ParamField path="scenarioId" type="integer" required>
  The scenario ID
</ParamField>

```bash
curl -X GET https://waterr.ai/backend/sentenced-transcripts/scenario/456 \
  -H "Authorization: Bearer YOUR_API_TOKEN"
```

{

## Get Sentence Transcripts for a Meeting

<ParamField path="transcript_id" type="integer" required>
  The transcript ID
</ParamField>

<ParamField path="meeting_id" type="integer" required>
  The meeting ID
</ParamField>

<ParamField path="scenario_id" type="integer" required>
  The scenario ID
</ParamField>

```bash
curl -X GET https://waterr.ai/backend/sentenced-transcripts/301/789/456 \
  -H "Authorization: Bearer YOUR_API_TOKEN"
```

```json
{
  "success": true,
  "data": {
    "sentences": [
      {
        "id": 1001,
        "transcript_id": 301,
        "speaker": "user",
        "message": "Hello, I'm excited to practice this interview.",
        "start_duration": "00:00",
        "sequence_number": 1
      }
    ],
    "meeting_info": { "id": 789 },
    "scenario_info": {
      "id": 456,
      "name": "Product Manager Interview Practice"
    }
  }
}
```

## Get Transcripts by Meeting ID (non-sentenced)

<ParamField path="meetingId" type="integer" required>
  The meeting ID
</ParamField>

```bash
curl -X GET https://waterr.ai/backend/transcripts/meeting/789 \
  -H "Authorization: Bearer YOUR_API_TOKEN"
```

```json
[
  {
    "id": 301,
    "meeting_id": 789,
    "content": "Full transcript text...",
    "status": "completed",
    "language": "en",
    "created_at": "2024-01-15T11:10:00Z"
  }
]
```

<Tip>
  If you're building product features on top of transcripts, prefer sentence transcripts because they're already normalized for parsing and ordering.
</Tip>
